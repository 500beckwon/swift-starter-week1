# 질문1 (컴퓨터가 십진법이 아닌 이진법을 사용하는 이유?)
이진법은 전기를 통해 연산한다. 
즉, 전기를 켜고 끄는 방식으로 작동하는 컴퓨터에 적합한 방법이다. 
이때 이진법은 두 가지의 회로, 즉, 0이냐 1이냐를 가지고 연산하기 때문에 십진법보다 훨씬 수월하게 자료를 처리할 수 있다. 
십진법을 컴퓨터에 적용하면 저장회로가 10개나 필요하며, 이는 컴퓨터가 관리해야 할 회로가 많아지는 것이기 때문에 이진법을 사용할 때에 비해 확률적으로 훨씬 더 많은 문제점이 발생한다는 사실을 알 수 있다.
따라서 오늘날까지 컴퓨터는 십진법이 아닌  0과 1만으로 우리만의 약속을 규정해 코드를 생성하고 있다.

# 질문2 ( week 0을 보면서 가장 인상 깊었던 것은 무엇이며, 그 이유는 무엇인가요?)
week 0을 보면서 가장 인상 깊었던 파트는 컴퓨터의 두 가지 구조에 대한 설명이었다.
**‘컴퓨터 이해하기’** 파트에서 컴퓨터의 구조는 하버드 구조와 폰 노이만 구조가 있다는 것을 알게 되었다. 
하버드 구조는 메모리 2개를 사용하고, 폰 노이만 구조는 메모리 1개를 사용한다. 
그렇다면 최신 컴퓨터들은 어떤 구조를 사용할까? 
답은 두 구조를 모두 사용한다는 것이다.
촤신 컴퓨터의 구조에서 cpu 내부는 하버드 구조, cpu 외부는 폰 노이만 구조로 되어있다는 사실을 이번 사전 과제를 공부하며 처음 알게 되었다.  
우리가 뇌에서 빠르게 계산을 처리할 수 있어도, 그 기억을 불러오는 속도가 느리면 전체적인 속도는 느려질 수밖에 없다. 
컴퓨터도 우리의 뇌와 같으므로 똑같은 문제점이 발생한다.
이런 현상이 컴퓨터에서 나타나는 문제점을 해결하기 위해 cpu가 명령어와 데이터를 동시에 사용할 수 있도록 물리적으로 구분한 하버드 구조를 cpu 내부에 적용한 것이다. 
이를 통해 최신 컴퓨터들의 속도가 많이 향상될 수 있었다. 
컴퓨터의 구조에 대해 막연한 궁금함만 있었던 나는 cpu의 내부와 외부의 cpu 구조가 다르다는 것이 매우 새롭고 참신하다고 느껴졌다. 
** 세계 곳곳에서는 항상 우리의 복지 를 개선하는 노력 (ex. 컴퓨터에서는 성능향상을 위한 노력) 을 하고 애쓰고 있다는 것이 나의 개발 의지를 더욱 불태웠다!) **
